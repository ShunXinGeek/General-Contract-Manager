# 通用合同管理助手 (General Contract Manager)

## 1. 程序简介
通用合同管理助手是一款基于 Web 的纯前端应用（支持离线运行 PWA 和云端同步），旨在为法律专业人士、企业法务及合同管理者提供高效、智能化、结构化的合同阅读、审阅、编辑及管理体验。通过引入 AI 大模型能力，实现了合同条款智能问答、知识库检索增强（RAG）以及双语辅助阅读等多种高级功能。

## 2. 功能特点
- **多开与独立状态管理**：支持同时打开多份合同（多标签页），各个标签页的工作区视图状态、引用面板收起/展开状态均相互独立，互不干扰。
- **智能化 AI 助手**：
  - 支持绑定自定义的大语言模型（如 GLM-4, DeepSeek 等），支持流式对话。
  - 支持配置 Embedding 和 Rerank 模型，内置 RAG 知识库机制以提高基于合同长文本的回答准确率。
  - "思考模式" 与 "知识库模式" 切换，支持一键引用合同条款精准向 AI 提问。
- **强大的阅读与辅助外设**：
  - **双重阅读模式**：中栏展示当前条款主干，右栏展示参考视图（Reference View）。支持在右侧面板一键查看条款的完整中文译文或引用内容。
  - **同步滚动**：针对双语版本，一键开启中英原文译文双屏自动同步精准对齐。
- **批注与行内富文本编辑**：
  - 提供轻量化的悬浮工具栏，选中文本即可调整字体颜色、高亮背景、加粗格式。
  - 支持针对条款或选段添加备注（Note）批注，以及使用书签（Bookmark）调整子条款层级（升级/降级）。
- **文档导入导出**：
  - 支持导入本地 JSON 修改数据备份以及标准 TXT 合同条款文件。
  - 可导出为 PDF，或导出为**带有审阅批注**的 Word (.docx) 文档以及无需批注的纯净版 Word。
- **云端同步与本地优先**：
  - 架构优先使用 LocalForage 存储文件数据，完全支持断网离线无缝使用。
  - 深度集成 Firebase，实现配置项与合同数据的云端备份、获取与多端自动同步。

## 3. 代码结构与组成
本系统采用轻量化 HTML/CSS/JS 结构搭建纯前端交互环境，无传统应用后端代码介入（依赖 Firebase 等第三方 BaaS）。
- `index.html`: 系统主界面入口。包含控制台欢迎页、左/中/右工作区面板、AI 助手聊天窗口以及系统设置模态框等 DOM 结构定义。
- `css/style.css`: 统揽整个应用的样式设计，包括交互动效、深色/护眼主题配色方案以及各个面板的布局参数。
- `js/` 目录（核心业务逻辑模块）:
  - **应用骨架:**
    - `app.js`: 页面全局状态控制、事件监听、欢迎页管理以及标签页（多开）调度与视图刷新。
    - `config.js`: 全局配置、系统常量、默认主题设定。
    - `utils.js` / `logger.js`: 抽离出的公共 DOM 操作脚手架与日志行为追踪工具类。
  - **AI 与检索模块:**
    - `ai-assistant.js`: 承接 AI 聊天交互、Prompt（系统提示词）注入以及与模型接口通信。
    - `rag.js`: 提供 Embedding（文本向量化）、知识库索引构建与检索逻辑增强。
    - `search.js`: 侧边栏的条款快速过滤功能以及正文搜索匹配实现。
  - **智能辅助与配置:**
    - `ai-settings.js`: 设置面板系统级交互，包含对话模型增删、API 参数设置以及系统行为配置保存。
    - `sync.js`: 核心双屏（原文/参考译文或中英对照）滚动同步运算引擎。
    - `comparison.js`: 特定情形下不同文本版本的变动差异（Diff）分析比对功能。
  - **阅读操作与云端:**
    - `editor.js`: 基于 Range 的文本富文本编辑管理、格式清理以及添加/管理浮动批注气泡（Notes）。
    - `bookmark.js`: 条款层级控制逻辑模块，包括特定书签标记与层级（Heading 级别）升级及降级。
    - `import.js`: 读取解析从用户设备上传的文件，并将页面上的富文本连同备注转译打包为 Blob 对象从而输出 Word 或 PDF。
    - `cloud-storage.js` / `firebase-config.js`: 操作 FirebaseAuth 及 Firestore，执行上云和云端拉取写入本地等逻辑。

## 4. 操作与使用说明
### 4.1 初始化与系统设置
1. 点击界面顶部 ⚙️ 按钮进入设置面板。
2. **AI 助手设置**：在对话模型卡片中新增并配置兼容 OpenAI 格式的大语言模型 API (需要 Endpoint 和 API Key)。您还可以定制系统提示词与 Embedding 等底层参数。
3. **Firebase 设置（进阶）**：如需多设备云同步，可在对应标签页中输入 Firebase Console 获取的 Project 参数；否则留空即可。

### 4.2 导入与合同管理
1. 在顶部导航栏点击 📥（导入）按钮，选择“导入合同条款 (.txt)”或是以前保存的“.json”数据。
2. 导入成功后，系统左侧将生成类似目录树的条款导航。
3. 点击欢迎页（管理控制台的第一个标签），在您导入的合同列表上触发悬停操作，您可以无缝点击合同名称直接原地重命名，或者点击悬浮的“删除”按钮进行缓存清理。

### 4.3 沉浸式阅读与批注审阅
1. 选中左侧导航某条款节点，在页面主区域中栏进行详情阅读。
2. **批注与排版**：用鼠标选中一段条款文本，松开后自动弹出悬浮工具栏。您可以使用颜色或高亮进行标注，也可以点击 📝 建立附带您个人意见的批注。
3. **右侧 Reference 视图**：对于配置了双语源的合同，点击主视图内的蓝色标题或者特殊引用链接，右侧即可展开针对该片段的完整译文/原文提示内容。
4. **AI 分析助手**：在合同标签页的右下侧可通过点击“管理助手”切换至 AI 面板。点击工具栏上的“引用条款”将自动把当前选定的法律文字带入对话框中，随即输入您的问题提交给 AI，例如：“请指出这段免责声明的法律风险”。

### 4.4 导出审阅文档
1. 当合同的颜色标记或者批注意见全部完成并确认无误后。
2. 点击顶栏 📤 （导出）按钮。
3. 点击“导出为带批注 Word (.docx)”，审阅结果将会被自动打包下载到本地计算机中，供未安装本系统的同事使用 Office 进行协同处理。

## 5. 注意事项
1. **浏览器兼容性**：因为使用了多项现代 Web API，推荐在最新版的 Google Chrome 或 Microsoft Edge 内核浏览器上访问使用，以获得完整的 PWA 离线支持。
2. **本地存储为王**：本程序为了最大化个人隐私保障，合同本体将通过 IndexedDB 完全存在于您的浏览器缓存中（除非开启了 Firebase 备份）。**请务必当心**：在执行“清理浏览器缓存”操作前，切记要先使用系统的导出 `.json` 功能备份未云化的高亮和批注数据，避免丢失。
3. **断网可用性**：应用开启后若网络意外断掉，页面将显示顶部黄色警告栏，此时无法访问 AI 请求及实时云端同步，但是已经加载的合同阅读、右侧对照面板、本地工具栏格式设置与导出完全不受影响。
4. **API 安全提醒**：您输入的所有第三方大模型 API Key 和系统级配置令牌都仅静态保存在您浏览器的本地存储库中，并不经由任何中间开发者服务器中转。因此在进行投屏等公开展现和屏幕录制时需留意隐藏 API 秘钥明文。
